Usage

The Index sheet will be automatically loaded even no action is done. It will load maximum last Constants.MaxMissingDaysToLoad days
Also its defined as trigger. See documentation

Load

Loads the data to Index sheet. Before loading it clears the data which is loaded previously based on selected date.

  • Yesterday: Loads the data for yesterday or first previous work day only.
  • This Week: Loads this week only. If today is Monday, will not load anything.

Clear

Clears the data from Index sheet based on selected date.

  • Yesterday: Clears the data from yesterday or first previous work day only.
  • This Week: Clears this week only. If today is Monday, will not clear anything.

Clear All: Removes all the data from Index sheet.

Scoring

Scoring is made by Score-Weights definitions. Each metric has its own range and weight on total score. So different perspectives can be applied.

Settings

The settings are saved under Constants.gs file.

  • Crossover.Url: Base url for XO API
  • Crossover.XAuthToken: The token value for the user. If its set, UserName/Password will not be used.
  • Crossover.UserName: The username for Crossover website
  • Crossover.Password: The password for Crossover website
  • MaxMissingDaysToLoad: Default: 14, Describes how many previous days will be checked for loading on document open. If any day missing within this range, it will load all of the date till yesterday.
  • Crossover.TrackerActivityGroupsTemplate: The value to get metrics from XO API. The default value is "/tracker/activity/groups?date={date}&fullTeam=true&groups=groups&managerId={managerId}&refresh=false&teamId={teamId}&weekly=false". If you want to load only ICs under this user (which is defined by Crossover.UserName), just change fullTeam=false in this url.
